# README for outputs/QA_reports/suggested_fixes/hits_reductions/

This directory contains the output of the `hits_reductions` data generated by the quarto QA template at `inst/extdata/_extensions/template_QA_veg_data.qmd`.

The filename syntax is as follows:

```
<type>_<base_input_file>_<date>_<short_SHA>.csv
```

where:

* `<type>` is the type of data being reported on (e.g. `hits_reductions`)
* `<base_input_file>` is the base input file used to generate the data
* `<date>` is the date the data was generated
* `<short_SHA>` is the short SHA of the git commit used to generate the data
  
This data identifies ground layer observations with a high number of hits that may need to be reduced. The output of hits_reductions is a table with the following columns:

* `survey`: The survey number
* `system`: The system name
* `waterway`: The waterway name
* `site`: The site name
* `transect`: The transect number
* `metres`: The metres along the transect
* `species`: The species name
* `ground_layer_species_hits`: The number of hits for the species in the ground layer
* `max_val`: The maximum value of hits across species within the ground layer (it should match `ground_layer_species_hits` for the species with the maximum number of hits)
* `total`: The total number of hits across all species in the ground layer
* `diff`: The difference between `max_val` and `total`
* `reduced_value`: A suggested value to reduce the number of hits for that species entry to.
